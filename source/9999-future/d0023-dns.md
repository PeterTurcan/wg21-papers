## Abstract

This paper asks the committee to advance an async name resolution type for coroutine-native I/O.

Without DNS, TCP sockets can only connect to hardcoded IP addresses. The `resolver` type bridges hostnames and services to the endpoints that transport sockets consume. One type, two overloads, one results range. The proposal is `std::io::resolver` - an awaitable name resolution facility that the kernel enforces. It is Paper 12 in the [Network Endeavor](https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2026/p4100r1.pdf)<sup>[1]</sup> series.

The companion paper *DNS: Design Rationale*<sup>[2]</sup> provides the platform mapping, the convergence record across six ecosystems, anticipated objections, and the implementation inventory. Read this paper for the proposal; read the companion when you need the audit trail.

This paper is Paper 12 in the series defined by [P4100R1](https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2026/p4100r1.pdf)<sup>[1]</sup>. It depends on Paper 1 (IoAwaitable Protocol, [P4003R3](https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2026/p4003r3.pdf)<sup>[3]</sup>) and Paper 11 (TCP, for `endpoint`, `ipv4_address`, and `ipv6_address`).

## 2. What This Paper Asks

The committee is asked to advance the following type for the standard library:

```cpp
namespace std::io {

  class resolver {
  public:
    class results_type;

    explicit resolver(execution_context& ctx);

    resolver(resolver const&) = delete;
    resolver& operator=(resolver const&) = delete;
    resolver(resolver&&) noexcept;
    resolver& operator=(resolver&&) noexcept;

    ~resolver();

    // Forward resolution: hostname + service to endpoints.
    IoAwaitable auto resolve(
        std::string_view host,
        std::string_view service);

    // Reverse resolution: endpoint to hostname.
    IoAwaitable auto resolve(endpoint const& ep);

    void cancel();
  };

}
```

One type. Two `resolve` overloads. One cancellation function.

## 3. The `resolver` Type

### 3.1 Construction

A resolver is constructed with a reference to an `execution_context` - the event loop that will deliver the completion.

```cpp
std::io::execution_context ctx;
std::io::resolver res(ctx);
```

The resolver holds a non-owning reference to the execution context. The execution context must outlive the resolver. The resolver is movable but not copyable.

### 3.2 `resolve()` - Forward Resolution

```cpp
IoAwaitable auto resolve(
    std::string_view host,
    std::string_view service);
```

Returns an awaitable that completes with a `results_type` - a range of endpoints matching the given hostname and service. The awaitable satisfies the IoAwaitable protocol defined in [P4003R3](https://www.open-std.org/jtc1/sc22/wg21/docs/papers/2026/p4003r3.pdf)<sup>[3]</sup>.

`host` is a hostname (`"example.com"`) or a numeric address string (`"192.168.1.1"`, `"::1"`). `service` is a service name (`"http"`) or a numeric port string (`"80"`). The resolver accepts both forms for both arguments.

```cpp
auto [ec, results] = co_await res.resolve(
    "example.com", "https");
if (!ec)
    for (auto const& entry : results)
        // entry.endpoint() is ready for tcp_socket::connect()
```

On failure, the result carries a `std::error_code` describing the resolution error - host not found, service unknown, network unreachable, or operation cancelled.

### 3.3 `resolve()` - Reverse Resolution

```cpp
IoAwaitable auto resolve(endpoint const& ep);
```

Returns an awaitable that completes with the hostname associated with the given endpoint. Reverse resolution maps an IP address and port back to a hostname and service name.

```cpp
auto [ec, results] = co_await res.resolve(remote_ep);
if (!ec)
    log("Connected to: ", results.host_name());
```

On failure, the result carries a `std::error_code`. Not every endpoint has a reverse mapping. The absence of a mapping is not an error in the transport sense - it is a DNS configuration that the resolver reports faithfully.

### 3.4 `cancel()`

```cpp
void cancel();
```

Cancels all outstanding `resolve()` operations on this resolver. Each cancelled operation completes with `std::errc::operation_canceled`. Cancellation is cooperative - the cancelled operation completes asynchronously through the normal completion path.

## 4. Resolution Results

### 4.1 `results_type`

```cpp
class resolver::results_type {
public:
    class iterator;
    using const_iterator = iterator;

    iterator begin() const;
    iterator end() const;
    bool empty() const;
    std::size_t size() const;

    std::string_view host_name() const;
    std::string_view service_name() const;
};
```

`results_type` is a forward range of resolution entries. Each entry provides an `endpoint` ready for use with `tcp_socket::connect()` or `udp_socket::bind()`.

A single hostname may resolve to multiple addresses - one IPv4 and one IPv6, or several addresses for load-balanced services. The range exposes all of them. The caller iterates and selects.

### 4.2 Iterator and Entry

Each element yielded by the `results_type` iterator provides:

```cpp
endpoint const& endpoint() const;
std::string_view host_name() const;
std::string_view service_name() const;
```

The endpoint contains the resolved address and port. The host and service names are the strings the resolver associated with the endpoint - for forward resolution, the original query strings; for reverse resolution, the names the DNS returned.

## 5. Forward Resolution

Forward resolution is the common path. A client application knows a hostname and a service. It needs endpoints.

```cpp
task<> connect_to_service(
    std::io::execution_context& ctx,
    std::io::tcp_socket& sock,
    std::string_view host,
    std::string_view service)
{
    std::io::resolver res(ctx);
    auto [ec, results] = co_await res.resolve(
        host, service);
    if (ec)
        throw std::system_error(ec);

    for (auto const& entry : results)
    {
        auto [ec2] = co_await sock.connect(
            entry.endpoint());
        if (!ec2)
            co_return;
    }
    throw std::system_error(
        std::make_error_code(
            std::errc::connection_refused));
}
```

The pattern - resolve, iterate, try each endpoint until one succeeds - is the standard algorithm for connecting to a hostname. The `results_type` range makes the pattern expressible as a range-for loop.

---

## 6. Reverse Resolution

Reverse resolution maps an endpoint back to a hostname. The primary use is logging and diagnostics.

```cpp
task<> log_peer(
    std::io::execution_context& ctx,
    endpoint const& remote)
{
    std::io::resolver res(ctx);
    auto [ec, results] = co_await res.resolve(remote);
    if (!ec)
        std::println("Peer: {}:{}",
            results.host_name(),
            results.service_name());
    else
        std::println("Peer: {} (reverse lookup failed)",
            remote);
}
```

Reverse resolution is not always available. Many network configurations do not maintain reverse DNS records. The resolver reports the absence through the error code. Application code that logs peer information should handle both paths.

## 7. Cancellation

Resolution can be cancelled explicitly or through the stop token propagated by the IoAwaitable protocol.

```cpp
task<> resolve_with_timeout(
    std::io::execution_context& ctx,
    std::string_view host,
    std::string_view service)
{
    std::io::resolver res(ctx);
    std::io::timer deadline(ctx, 5s);

    auto result = co_await when_any(
        res.resolve(host, service),
        deadline.wait());

    if (result.index() == 1)
        throw std::system_error(
            std::make_error_code(std::errc::timed_out));
}
```

Explicit cancellation via `cancel()` cancels all outstanding operations on the resolver. The stop token path cancels individual operations. Both paths deliver `std::errc::operation_canceled` to the awaiting coroutine.

## 8. Example Usage

### 8.1 Resolve and Connect

The most common pattern in networked applications:

```cpp
task<> http_client(
    std::io::execution_context& ctx,
    std::string_view host)
{
    std::io::resolver res(ctx);
    auto [ec, endpoints] = co_await res.resolve(
        host, "https");
    if (ec)
        throw std::system_error(ec);

    std::io::tcp_socket sock(ctx);
    for (auto const& entry : endpoints)
    {
        auto [ec2] = co_await sock.connect(
            entry.endpoint());
        if (!ec2)
        {
            co_await do_http_request(sock);
            co_return;
        }
    }
    throw std::runtime_error("no reachable address");
}
```

### 8.2 Dual-Stack Resolution

```cpp
task<> connect_dual_stack(
    std::io::execution_context& ctx,
    std::string_view host,
    std::string_view service)
{
    std::io::resolver res(ctx);
    auto [ec, results] = co_await res.resolve(
        host, service);
    if (ec)
        throw std::system_error(ec);

    // Try IPv6 first, then IPv4.
    for (auto const& entry : results)
    {
        std::io::tcp_socket sock(ctx);
        auto [ec2] = co_await sock.connect(
            entry.endpoint());
        if (!ec2)
        {
            co_await handle_connection(sock);
            co_return;
        }
    }
}
```
